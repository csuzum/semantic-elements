HTML5 Semantic Elements Summarized

What is Divitis?

Divitis is a common problem with web designers. It is the process of using too many nested/unnecessary divs to mark up a page.

Semantic elements clearly describe their meaning to both browsers and developers. The elements div and span are essentially generic in content, lack a clear definition and we have better alternatives now. For example, instead of div with IDs and classes, HTML5 uses elements such as section, aside, table, form, and article that make the intent of the components more apparent. These more clearly defined semantic tags make the content more accessible, increase searchability, better understood by non-native English speakers, and help other programmers understand the structure. HTML is about organization of the content. Elements previously used to describe styling have been depreciated from use in HTML. Instead, styling and presentational elements such as center, font, small, footer, and img should be handled in CSS instead of in HTML.
